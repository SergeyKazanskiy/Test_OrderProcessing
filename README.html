📡 Event Bus

Реалізована in-process async Event Bus (app/events/bus.py) на базі asyncio.Task.

  • publish() — fire-and-forget: планує кожен обробник як окремий asyncio.Task
  • Видавець не блокується і не чекає завершення обробників
  • Падіння одного підписника ізольоване — інші обробники продовжують роботу
  • Підтримуються як async def, так і звичайні def хендлери
  • Декоратор @bus.on("event.type") для зручної реєстрації
# Публікація події в сервісі
await event_bus.publish(
    Event(
        event_type=OrderEvents.CREATED,
        payload={"order_id": order.id, "total_amount": str(order.total_amount)},
    )
)

# Підписка через декоратор
@event_bus.on("order.created")
async def on_order_created(event: Event) -> None:
    logger.info("New order: %s", event.payload["order_id"])
ПодіяТригер
customer.createdСтворення нового клієнта
product.createdСтворення нового товару
order.createdОформлення нового замовлення

⚖️ Виділений Rules Service

Бізнес-правила винесені в окремий stateless-шар (app/rules/order_rules.py).

  • Правила змінюються незалежно від логіки персистентності
  • Легко тестувати як чисті функції — без БД, без HTTP
  • Сервіси викликають Rules до будь-якої мутації даних
# В OrderService — одна точка входу для всіх правил
OrderRules.validate_order_create(dto, customer, found_products)

🌍 Локалізація відповідей

Всі повідомлення API (успіх і помилки) повертаються мовою клієнта.

  1. FastAPI-залежність get_locale розбирає заголовок Accept-Language
  2. Обирається найкращий збіг із підтримуваних локалей (en, uk, ru)
  3. Функція translate(key, locale, **context) підставляє змінні в шаблон
  4. При невідомій локалі — fallback на default_locale (за замовчуванням en)
Accept-Language: uk   →  "Замовлення успішно створено."
Accept-Language: ru   →  "Заказ успешно создан."
Accept-Language: fr   →  "Order created successfully."  (fallback)

💰 Автоматичний розрахунок суми

Метод Order.recalculate_total() обчислює суму на рівні доменної моделі. unit_price знімається як знімок з Product.price у момент створення замовлення.

def recalculate_total(self) -> None:
    self.total_amount = sum(
        (item.unit_price * item.quantity for item in self.items),
        Decimal("0.00"),
    )

🔒 Бізнес-правилаБизнес-правила

Всі правила перевіряються до будь-якого звернення до бази даних.

👤 Клієнт

  • Email повинен бути унікальним в системі

📦 Товар

  • SKU повинен бути унікальним в системі
  • Ціна повинна бути строго позитивною

🧾 Замовлення

  • Не можна створити замовлення без існуючого клієнта
  • Замовлення повинно містити хоча б один товар
  • Кількість кожного товару в замовленні — позитивне ціле число
  • Всі product_id в позиціях замовлення повинні існувати в БД
  • Сума замовлення розраховується автоматично і не приймається від клієнта

🌍 ЛокалізаціяЛокализация

Для отримання відповіді потрібною мовою передавайте заголовок Accept-Language:

Accept-Language: uk          # Українська
Accept-Language: ru          # Русский
Accept-Language: en          # English (default)
Accept-Language: uk,en;q=0.9 # Українська з fallback на англійську

Додавання нової мови — достатньо додати словник у app/i18n/translator.py та включити код локалі в SUPPORTED_LOCALES.
